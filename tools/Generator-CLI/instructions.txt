Generator-CLI - Usage Instructions
===================================

Generator-CLI is a command-line tool designed to combine selected RFC framework modules and their dependencies into a single header and C++ source file pair. This facilitates modular builds and decreases compilation times.

Usage:
  Generator-CLI.exe [options] [modules...]

Options:
  -rfc-dir, -r   Path to the RFC root directory (containing core, gui, etc.).
                 If not specified, the tool will attempt to auto-detect the path
                 by checking current and parent directories.
  -out-dir, -o   Directory to write the amalgamated C++ header and source files.
                 Defaults to the current directory (".").
  -name, -n      Base name of the amalgamated files. Generates <name>.h and
                 <name>.cpp. Defaults to "rfc".
  -modules, -m   Comma-separated list of modules to include.
  -list, -l      List all available modules, dependencies, and OS compatibility.

Examples:
  1. List all available modules:
     Generator-CLI.exe -l

  2. Combine "gui", "file", and "thread" modules, outputting to "include" directory:
     Generator-CLI.exe gui file thread -o ./include

  3. Build with custom output file names (e.g. custom_rfc.h and custom_rfc.cpp):
     Generator-CLI.exe -m gui,file -n custom_rfc

  4. Run with explicit RFC path:
     Generator-CLI.exe gui file -r C:\Projects\rfc_test\rfc -o ./output

Note:
  - The "Core" module is fixed (always-on) and is automatically included.
  - All recursive dependencies for selected modules are automatically resolved
    and included.
